    Examples¶

    Complete worked examples covering common simulation scenarios.

    Example 1 — Static GPS L1 (Simplest Case)¶

    A stationary receiver at a fixed location, receiving GPS L1 C/A signals from all visible satellites.

    Scenario Config¶

    config/example_static_l1.yaml
    scenario:
  start_epoch_utc: "2026-01-15T12:00:00Z"
  duration_s: 30.0
  rx_latitude: 32.08
  rx_longitude: 34.78
  rx_altitude_m: 50.0
  elevation_mask_deg: 10.0

  satellites:
    rinex_nav_file: null
    constellations:
      GPS_L1:
        enabled: true
        common_power_dbm: -130.0

  antenna_array:
    elements:
      - id: "ANT_1"
        lever_arm: [0.0, 0.0, 0.0]
        orientation: [0.0, 0.0, 0.0]

    What Happens¶

    1. Server downloads RINEX navigation data for January 15, 2026
    2. Computes which GPS satellites are visible from Tel Aviv (≈8–12 SVs typically)
    3. Generates 300 frames (30s ÷ 100ms) of GPS L1 C/A signal at 6.138 MHz
    4. Client adds thermal noise, quantizes, and writes a single .bin file
    5. PocketSDR acquires and tracks all visible satellites

    Expected Output¶

    [SERVER] Found 10 visible satellites above 10 deg elevation.
Total: 45.2s for 300 frames (150ms/frame avg)
--- PocketSDR Tracking Results ---
PRN 01: CN0=42.3 dBHz, Doppler=-1523 Hz
PRN 03: CN0=39.8 dBHz, Doppler=+2104 Hz
...

    Example 2 — Dynamic Dual-Band with Trajectory¶

    A moving receiver tracking GPS L1 + L2 simultaneously.

    Scenario Config¶

    config/example_dynamic_dualband.yaml
    scenario:
  start_epoch_utc: "2026-01-15T08:00:00Z"
  duration_s: 60.0
  receiver_trajectory: "experiments/trajectories/dynamic_test.csv"
  elevation_mask_deg: 10.0

  satellites:
    rinex_nav_file: null
    constellations:
      GPS_L1:
        enabled: true
        common_power_dbm: -130.0
      GPS_L2:
        enabled: true
        common_power_dbm: -130.0

  antenna_array:
    elements:
      - id: "ANT_1"
        lever_arm: [0.0, 0.0, 0.0]
        orientation: [0.0, 0.0, 0.0]

    What Happens¶

    1. Server loads the trajectory CSV and evaluates 6-DoF dynamics at each frame
    2. Both L1 and L2 signals are generated simultaneously on the GPU
    3. Output contains interleaved dual-band data in RAW8 format
    4. PocketSDR tracks both L1CA and L2CM signals

    Key Differences from Static¶

    Aspect Static Dynamic
    Receiver position Fixed ECEF from LLA Evaluated from trajectory polynomials
    Doppler Satellite motion only Satellite + receiver motion
    Output format INT8X2 (single band) RAW8 (dual band interleaved)
    PocketSDR signals L1CA only L1CA + L2CM

    Example 3 — Jamming Scenario¶

    A static receiver under attack from a CW jammer and a broadband noise jammer.

    Scenario Config¶

    config/example_jamming.yaml
    scenario:
  start_epoch_utc: "2026-01-15T12:00:00Z"
  duration_s: 30.0
  rx_latitude: 32.08
  rx_longitude: 34.78
  rx_altitude_m: 50.0
  elevation_mask_deg: 10.0

  satellites:
    rinex_nav_file: null
    constellations:
      GPS_L1:
        enabled: true
        common_power_dbm: -130.0

  interference:
    emitter_file: "config/jammers_demo.csv"

  antenna_array:
    elements:
      - id: "ANT_1"
        lever_arm: [0.0, 0.0, 0.0]
        orientation: [0.0, 0.0, 0.0]

    Jammer Definition¶

    config/jammers_demo.csv
    id,type,geometry_mode,latitude_deg,longitude_deg,altitude_m,power_dbm,center_freq_hz,bandwidth_hz,freq_offset_hz
JAM_CW,CW,STATIONARY_LLA,32.0853,34.7818,50.0,-40.0,1575.42e6,0,200.0
JAM_BB,BROADBAND_NOISE,STATIONARY_LLA,32.0900,34.7750,30.0,-30.0,1575.42e6,20.0e6,

    What Happens¶

    1. Two jammers are loaded: a CW jammer at -40 dBm and a broadband noise jammer at -30 dBm
    2. Jammer signals are generated on the GPU alongside satellite signals
    3. Both are summed into the same output buffer — the composite received signal
    4. PocketSDR attempts to acquire and track through the interference

    Expected Impact¶

    Metric Clean With Jammers
    CN0 (typical) 42 dB-Hz Degraded (depends on J/S ratio)
    Satellites tracked 8–12 Reduced
    Doppler accuracy < 1 Hz May be degraded

    Jammer-to-Signal Ratio

    The CW jammer at -40 dBm is 90 dB above the GPS signal at -130 dBm. This represents a strong in-band interference scenario. The broadband jammer at -30 dBm spreads its energy across 20 MHz, so the in-band J/S ratio depends on the receiver bandwidth.

    Example 4 — Multi-Element Antenna Array (CRPA)¶

    A 4-element array for adaptive beamforming / null-steering research.

    Scenario Config¶

    config/example_crpa.yaml
    scenario:
  start_epoch_utc: "2026-01-15T12:00:00Z"
  duration_s: 30.0
  rx_latitude: 32.08
  rx_longitude: 34.78
  rx_altitude_m: 50.0
  elevation_mask_deg: 10.0

  satellites:
    rinex_nav_file: null
    constellations:
      GPS_L1:
        enabled: true
        common_power_dbm: -130.0

  interference:
    emitter_file: "config/jammers_demo.csv"

  antenna_array:
    elements:
      - id: "ANT_1"
        lever_arm: [0.0, -0.05, 0.0]
        orientation: [0.0, 0.0, 0.0]
      - id: "ANT_2"
        lever_arm: [0.0, 0.05, 0.0]
        orientation: [0.0, 0.0, 0.0]
      - id: "ANT_3"
        lever_arm: [0.05, 0.0, 0.0]
        orientation: [0.0, 0.0, 0.0]
      - id: "ANT_4"
        lever_arm: [-0.05, 0.0, 0.0]
        orientation: [0.0, 0.0, 0.0]

    What Happens¶

    1. Server generates 4 independent IQ streams (one per element), each with unique propagation delays reflecting the element's physical offset
    2. All streams contain the same satellite signals and jammer signals — but with element-specific phase shifts due to lever arm geometry
    3. Client writes 4 separate .bin files: ant0.bin, ant1.bin, ant2.bin, ant3.bin
    4. A downstream adaptive algorithm (e.g., MVDR, null-steering) can combine these streams to suppress the jammer while preserving satellite signals

    Use Case¶

    This is the primary use case for the simulator: generating the multi-channel data required to develop and validate Controlled Reception Pattern Antennas (CRPA) and adaptive anti-jam algorithms. The simulator provides:

    • Ground truth phase-coherent signals across all elements
    • Controllable jammer scenarios (position, type, power)
    • Repeatable test conditions (deterministic simulation)
