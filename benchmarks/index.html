    Performance Benchmarks¶

    This page tracks the runtime performance of the GNSS Testbed Simulator across various scenarios based on the latest runs.

    O(1) Frame Processing

    Each frame is processed in constant time regardless of total simulation duration. The values below represent per-frame processing times. Total simulation time scales linearly: total_time = frames × time_per_frame.

    🚀 Recent Improvements (126a115 → 49000f9)¶

    The transition to the new Blackwell architecture (49000f9) provided massive server-side frame generation speedups across all basic L1 configurations (measured at 6.1 MHz, 0 jammers):

    Scenario Server ms (126a115) Server ms (49000f9) Performance Gain Total ms (49000f9)
    1 antenna 14.32 8.84 38% faster 17.76
    4 antennas 30.94 17.89 42% faster 53.48
    8 antennas 51.41 28.09 45% faster 133.02
    16 antennas 93.42 49.82 46% faster 402.85

    Note: All detailed metrics below are derived from the latest 49000f9 build running on NVIDIA RTX PRO 4500 Blackwell.

    Timing Definitions¶

    Metric Description
    Server Processing GPU frame generation time (geometry + signal rendering + jammer injection)
    Communication ZeroMQ transfer time (server → client), including serialization overhead
    Client Processing Noise addition + AGC + quantization + file write
    Total End-to-end per-frame latency

    Slowdown Factor

    The slowdown factor is defined as frame_processing_time / frame_duration. A slowdown of ×10 means 100 ms of simulated signal takes 1,000 ms to generate. Lower is better.

    How to Measure¶

    To collect timing data for a specific scenario, run the client with the desired configuration and observe the per-batch timing output:

    [1/60] Requesting batch... server=1523ms  proc=342ms
[2/60] Requesting batch... server=1518ms  proc=339ms
...
Total: 112.3s for 600 frames (187ms/frame avg)

    The server time includes GPU processing + ZMQ transfer. The proc time is client-side post-processing. Divide server time by n_frames per batch to get per-frame server time.
