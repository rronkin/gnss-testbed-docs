    Configuration Reference¶

    The simulator is driven by two separate YAML files, each targeting a different audience and change frequency.

    File Role Audience Changes
    scenario_config.yaml What to simulate User / test engineer Per scenario
    simulator_params.yaml How to simulate System designer Rarely

    Scenario Configuration¶

    File: config/scenario_config.yaml

    This file defines the simulation scenario: time window, receiver position/trajectory, satellite constellations, interference, and antenna array.

    Full Annotated Example¶

    scenario_config.yaml
    scenario:
  # ── Time ──
  start_epoch_utc: "2026-01-15T08:00:00Z"   # Simulation start (ISO 8601 UTC)
  duration_s: 90.0                            # Total simulation duration (seconds)

  # ── Receiver Position ──
  # Option A: Static position (LLA)
  rx_latitude: 32.08                          # WGS-84 latitude (degrees, north +)
  rx_longitude: 34.78                         # WGS-84 longitude (degrees, east +)
  rx_altitude_m: 50.0                         # Height above ellipsoid (meters)
  rx_orientation: [0.0, 0.0, 0.0]            # [yaw, pitch, roll] in degrees

  # Option B: Dynamic trajectory (overrides static position)
  # receiver_trajectory: "path/to/trajectory.csv"

  elevation_mask_deg: 10.0                    # Min satellite elevation (degrees)

  # ── Satellites ──
  satellites:
    rinex_nav_file: null                       # null = auto-download from CDDIS
    constellations:
      GPS_L1:
        enabled: true
        common_power_dbm: -130.0               # Nominal received power (dBm)
        sv_power_offsets_db:                    # Optional per-SV delta (dB)
          G01: +2.0
          G15: -1.5
      GPS_L2:
        enabled: true
        common_power_dbm: -130.0
      Galileo_E1:
        enabled: false
        common_power_dbm: -130.0

  # ── Interference ──
  interference:
    emitter_file: "config/jammers_demo.csv"    # Path to jammer CSV (omit to disable)

  # ── Antenna Array ──
  antenna_array:
    elements:
      - id: "ANT_1"
        lever_arm: [0.0, -0.05, 0.0]          # 5 cm left of COM [fwd, right, down]
        orientation: [0.0, 0.0, 0.0]           # [yaw, pitch, roll] degrees
        pattern: "isotropic"                   # or path to CST pattern file

      - id: "ANT_2"
        lever_arm: [0.0, 0.05, 0.0]           # 5 cm right of COM
        orientation: [0.0, 0.0, 0.0]

      - id: "ANT_3"
        lever_arm: [0.05, 0.0, 0.0]           # 5 cm forward
        orientation: [0.0, 0.0, 0.0]

      - id: "ANT_4"
        lever_arm: [-0.05, 0.0, 0.0]          # 5 cm aft
        orientation: [0.0, 0.0, 0.0]

    Scenario Parameters¶

    Parameter Type Unit Default Description
    start_epoch_utc string ISO 8601 2026-01-15T12:00:00Z Simulation start time
    duration_s float seconds 60.0 Total simulation duration
    receiver_trajectory string path null CSV trajectory file (see Inputs)
    rx_latitude float degrees 48.8584 Static receiver latitude (WGS-84)
    rx_longitude float degrees 2.2945 Static receiver longitude (WGS-84)
    rx_altitude_m float meters 330.0 Static receiver altitude
    rx_orientation float[3] degrees [0,0,0] Static receiver attitude [yaw, pitch, roll]
    elevation_mask_deg float degrees 10.0 Minimum satellite elevation to include

    Constellation Parameters¶

    Parameter Type Default Description
    enabled bool false Enable this constellation-band
    common_power_dbm float -130.0 Nominal received power for all SVs (dBm)
    sv_power_offsets_db map {} Per-SV power offset (dB), additive

    Antenna Element Parameters¶

    Parameter Type Unit Default Description
    id string — "ANT_1" Unique element identifier
    lever_arm float[3] meters [0,0,0] Offset from COM in body frame [forward, right, down]
    orientation float[3] degrees [0,0,0] Boresight direction [yaw, pitch, roll] (Z-Y-X)
    pattern string — "isotropic" "isotropic" or path to antenna pattern file

    Simulator Parameters¶

    File: config/simulator_params.yaml

    This file controls the simulation engine's internal settings. It is typically configured once per hardware profile and rarely changed between scenarios.

    Full Annotated Example¶

    simulator_params.yaml
    simulator:
  # ── Timing ──
  frame_period_s: 100e-3                      # Frame duration (outer loop period)
  channel_period_s: 1e-3                       # Geometry snapshot cadence

  # ── Sampling ── (per RF band)
  sample_rate_hz:
    L1: 6.138e6                                # L1 band sample rate (Hz)
    L2: 6.138e6                                # L2 band sample rate (Hz)

  carrier_freq_hz:
    L1: 1575.42e6                              # L1 carrier frequency (Hz)
    L2: 1227.60e6                              # L2 carrier frequency (Hz)

  # ── Polyphase Resampler ──
  polyphase_L: 64                              # Number of polyphase branches
  polyphase_taps: 24                           # Filter depth per branch
  polyphase_beta: 8.0                          # Kaiser window beta
  polyphase_cutoff_frac: 0.85                  # Prototype filter cutoff (fraction of Nyquist)

pipeline:
  enable_telemetry: true                       # Write per-frame telemetry
  batch_size: 10                               # Emitters processed concurrently

debug:
  freeze_nav_bit: false                        # Force nav bit to +1 (debug)
  freeze_dynamics: false                       # Force zero delay/Doppler (debug)

    Simulator Parameters¶

    Parameter Type Unit Default Description
    frame_period_s float seconds 0.1 Duration of each processing frame
    channel_period_s float seconds 0.001 Coarse delay interpolation cadence
    sample_rate_hz float or dict Hz 50.0e6 Sampling rate (scalar or per-band)
    carrier_freq_hz float or dict Hz 1575.42e6 Carrier frequency (scalar or per-band)
    noise_power_dbm float dBm/Hz -174.0 Thermal noise power spectral density
    polyphase_L int — 64 Polyphase filter bank branches
    polyphase_taps int — 24 Taps per polyphase branch
    polyphase_beta float — 8.0 Kaiser window shape parameter
    polyphase_cutoff_frac float — 0.85 Filter cutoff fraction

    Timing Constraints¶

    The following relationships must hold:

    \[T_s \ll T_\text{channel} \leq T_\text{frame}\]
    • \(T_\text{frame} / T_\text{channel}\) must be an integer (exact number of snapshots per frame)
    • \(T_\text{channel} \times F_s\) must be an integer (exact number of samples per snapshot)
